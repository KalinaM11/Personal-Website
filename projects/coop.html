Co-op Application Website

This project is a data-driven co-op application website that supports five different types of users: Students, Companyb workers, Advisors, Alumni, and System Administrators. You can "sign in" to each persona by click on "Act as ___". Each persona has 3 pages, which each support essential needs for that persona. A rest API was used to link together the backend SQL database (through SQL queries) and the front-end streamlit UI. We created our own Relational database in SQL.

Link to github repo

Features

  • A student currently taking classes at Northeastern can apply to co-ops and use special filters, connect with alumni and their advisor.
  • An advisor can interact with their students by viewing their applications. Should also be able to filter students that have and have not recieved co-ops and view their profiles.
  • An alumni who has graduated from Northeastern having done a co-op should be able to connect with students to give them advice. They should also be able to display thier previous co-op experiences.
  • A system administrator can view tickets of errors that need fixing, resolve or delete these errors and they should have full control over all users including, deleting accounts.

Technologies Used

Python, Streamlit for UI, SQL, Docker, VS Code, GitHub
